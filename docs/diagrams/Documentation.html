🛡️ Introduction

NightOwl is an autonomous Site Reliability Engineering (SRE) platform that leverages AI agents to handle the full incident lifecycle — from detection to resolution and documentation — reducing MTTR from 45+ minutes to 3-5 minutes.

Key Value: NightOwl replaces manual on-call toil with intelligent, autonomous agents that collaborate in real-time to diagnose and fix production incidents.

Core Capabilities

  • Automated Detection: Ingests alerts from Grafana, GitHub, Kubernetes, and Slack
  • AI-Powered Diagnosis: Multi-agent system using CrewAI for root cause analysis
  • Autonomous Remediation: Executes fixes via MCP tools (restart, rollback, scale)
  • Auto-Documentation: Generates post-mortems with timelines and RCA
  • Real-time Dashboard: React UI with Socket.io for live incident tracking
  • Compliance & Security: SOC2/GDPR auditing and CVE scanning

🤖 AI Agents (CrewAI Swarm)

NightOwl uses a multi-agent architecture powered by CrewAI. Each agent has a specialized role, specific tools, and collaborates with other agents to handle incidents end-to-end.

🔍 Triage Agent

First responder. Assesses severity (critical/warning/info), sets priority, and routes incidents to the appropriate investigation agents.

prometheus_query

📄 Log Analyst

Parses logs from Loki, identifies error patterns and frequencies, and performs root cause analysis from log data.

loki_query

📈 Metric Correlator

Queries Prometheus time-series data, correlates metrics across services, and detects anomalies that coincide with incidents.

prometheus_query

🔧 Fixer Agent

Executes remediation: restart deployments, rollback releases, scale resources. Verifies the fix by checking pod health.

k8s_restartk8s_rollbackk8s_scale

🛡️ Security Auditor

Performs static analysis (SAST), CVE detection, and secret scanning on codebases and container images.

github_scan

✅ Compliance Checker

Validates infrastructure against SOC2, GDPR, and custom policies. Detects configuration drift from desired state.

terraform_plan

📝 PostMortem Writer

Generates detailed incident reports with root cause, impact, timeline, resolution steps, and follow-up action items.

report_gen

💾 Data Layer

🐘 PostgreSQL :5432

Primary database. Stores incidents, agent executions, remediations, and full audit trail. Uses UUID primary keys and JSONB for flexible metadata.

⚡ Redis :6379

In-memory cache for agent state, user sessions, and pub/sub for real-time event broadcasting across services.

🧲 Qdrant :6333

Vector database for log embeddings. Enables similarity search to find related past incidents and learn from historical resolutions.

📊 Prometheus :9090

Time-series metrics store. Scrapes metrics from all services (agent-gateway, event-dispatcher, mcp-server) every 15s.

🚀 Quick Start

Prerequisites

  • Docker & Docker Compose installed
  • Git installed
  • OpenRouter API key (free tier available)

Setup

# 1. Clone the repository
git clone https://github.com/your-repo/nightowl.git
cd nightowl

# 2. Configure environment
cp .env.example .env
# Edit .env with your API keys

# 3. Start all services
docker-compose up -d

# 4. Access the platform
# Dashboard:        http://localhost:3000
# API Gateway:      http://localhost:8000
# Event Dispatcher: http://localhost:3001
# Grafana:          http://localhost:3002 (admin/admin)

🗄️ Database Schema

incidents

CREATE TABLE incidents (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    alert_id        VARCHAR(255),
    source          VARCHAR(50) NOT NULL,     -- 'grafana', 'github', 'k8s'
    severity        VARCHAR(20) NOT NULL,     -- 'critical', 'warning', 'info'
    status          VARCHAR(30) DEFAULT 'open',
    title           TEXT NOT NULL,
    service         VARCHAR(255),
    description     TEXT,
    root_cause      TEXT,
    resolution      TEXT,
    auto_resolved   BOOLEAN DEFAULT FALSE,
    resolved_by     VARCHAR(255),
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    resolved_at     TIMESTAMPTZ,
    metadata        JSONB DEFAULT '{}'
);

agent_executions

CREATE TABLE agent_executions (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    incident_id     UUID REFERENCES incidents(id) ON DELETE CASCADE,
    agent_name      VARCHAR(100) NOT NULL,
    input_data      JSONB,
    output_data     JSONB,
    tool_calls      JSONB,
    tokens_used     INTEGER,
    duration_ms     INTEGER,
    status          VARCHAR(20),
    error_message   TEXT,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

remediations

CREATE TABLE remediations (
    id              UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    incident_id     UUID REFERENCES incidents(id) ON DELETE CASCADE,
    action_type     VARCHAR(50) NOT NULL,
    target          VARCHAR(255) NOT NULL,
    namespace       VARCHAR(100),
    status          VARCHAR(20),
    before_state    JSONB,
    after_state     JSONB,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

Indexes

CREATE INDEX idx_incidents_status     ON incidents(status);
CREATE INDEX idx_incidents_created    ON incidents(created_at DESC);
CREATE INDEX idx_incidents_service    ON incidents(service);
CREATE INDEX idx_incidents_severity   ON incidents(severity);
CREATE INDEX idx_agent_exec_incident  ON agent_executions(incident_id);
CREATE INDEX idx_remediations_incident ON remediations(incident_id);

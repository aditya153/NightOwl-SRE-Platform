NightOwl: Technical Architecture Specification

Comprehensive technical documentation detailing the networking, infrastructure, data structures, asynchronous programming patterns, and AI integration for an autonomous Agentic Site Reliability Engineering (SRE) platform.
1. UI & Observability Layer (Frontend)

Frontend Architecture

React 18 Vite Tailwind CSS TanStack Query v5 Socket.io-client React Router DOM

Component & State Management

Designed as a high-performance Single Page Application (SPA) prioritizing real-time observability over traditional CRUD operations.

  • Asynchronous State (TanStack Query): Utilized for aggressive caching, background refetching, and pagination of 1000+ historical incidents. Eliminates traditional Redux boilerplate while ensuring the UI is strictly synchronized with the Python Backend API.
  • Component Structure: Modular functional components using React Hooks (useState, useEffect). Separate contextual views for the Master Incident Dashboard, Collapsible AI Agent Status Sidebars, and Interactive Chat-Style Reasoning logs.

Real-time Telemetry & Styling

Built to handle heavy, rapid telemetry updates without choking the browser's render cycle.

  • WebSocket Streaming (Socket.io): Establishing persistent TCP connections to the backend. When a new Kafka event triggers an AI triage, the backend emits granular, chunked updates to the UI, allowing SREs to watch the AI "think" in real-time.
  • Utility-First Styling (Tailwind): Implements a custom dark-mode design system with strict color tokens. Includes complex CSS Grid and Flexbox layouts for dashboards, syntax-highlighted code blocks for raw JSON payloads, and micro-animations for UX polish.
2. API & Event Bus Layer (Backend)

Backend Microservices

Node.js / Express Python 3.11 / FastAPI KafkaJS & aiokafka Uvicorn asyncio

The Ingestion Layer (Node.js Dispatcher)

A lightweight, purely asynchronous ingestion API explicitly designed to never drop incoming webhooks during high traffic.

  • Express.js Entrypoint: Handles raw POST payloads from GitHub (Issues, PRs) and Grafana (Prometheus Alerts).
  • Kafka Producer Hook (KafkaJS): Validates payloads and immediately pushes them to the alerts-topic Kafka partition.
  • Non-blocking I/O: Returns a 200 OK response to the webhook provider instantly before the complex AI processing begins, preventing timeout errors on the client side during severity spikes.

The Orchestration Layer (Python Gateway)

A heavy-compute FastAPI server managing all asynchronous database I/O and synchronous machine learning inference.

  • Kafka Consumer (aiokafka): Operates an infinite async for msg in consumer loop listening to the alerts-topic. Guaranteed at-least-once delivery semantics for all incidents.
  • Thread Pool Offloading: Because LLM API requests and CrewAI execution are inherently synchronous (blocking operations), they are passed to asyncio.get_event_loop().run_in_executor(). This ensures the main Kafka consumer loop continues pulling new events while the AI is busy "thinking" on a separate CPU thread.
  • REST Architecture: Uses Pydantic for rigid request/response model validation and serialization for the Frontend.
3. Hybrid Persistence Layer (Database)

Dual Database Strategy (ACID + Vector)

PostgreSQL 15 SQLAlchemy (asyncpg) Alembic Qdrant HuggingFace Embeddings

Relational OLTP (PostgreSQL)

The immutable ledger for system events. Focused on data integrity, foreign key constraints, and fast localized writes.

  • Asynchronous ORM: Engine configured via postgresql+asyncpg:// to allow non-blocking database queries concurrent with the FastAPI server load.
  • Schema Design: incidents table utilizes native PostgreSQL JSONB columns to handle highly variable incoming webhook payloads without strict column refactors. agent_logs table utilizes ondelete='CASCADE' foreign keys to maintain referential integrity.
  • Migrations: Automated schema version control via Alembic (`alembic upgrade head`), ensuring reproducible CI/CD environments.

Retrieval Augmented Generation (RAG Memory)

A long-term semantic memory architecture allowing the AI agents to leverage historical playbooks and prior automated resolutions.

  • Vector Database (Qdrant): Deployed locally via Docker. Exposes a fast gRPC/REST interface for massive similarity searches.
  • Local Embeddings Pipeline: Integrated with langchain and sentence-transformers (HuggingFace all-MiniLM-L6-v2). Operates 100% locally on CPU to convert complex Agent reasoning text into 384-dimensional mathematical arrays.
  • Cosine Similarity Match: When a new alert hits the Kafka pipeline, the payload is embedded and compared against Qdrant via Cosine Similarity. The top 3 closest historical matches are injected directly into the Triage Agent's prompt context window.
4. Agentic AI & Autonomy (Machine Learning)

Multi-Agent Workflow & execution

CrewAI LangChain Google Gemini 2.0 Flash Model Context Protocol (MCP) Slack API (HITL)

Sequential Orchestration

A highly customized adaptation of the CrewAI framework designed for linear, fault-tolerant infrastructure diagnostics.

  • Triage Agent: Parses raw JSON payloads against RAG memory to assign a standardized Severity Score (Critical, High, Medium, Low).
  • Log Analyst Agent: Utilizes specialized regex prompts and token-optimized context windows to parse noisy Docker/Kubernetes stack traces.
  • Correlator Agent: Deduplicates concurrent Kafka events (e.g., matching a database timeout alert with a high CPU alert).
  • Compliance Agent: Hard-coded guardrails that validate proposed fixes against standard security policies before execution.

Standardized Tooling (MCP)

Implementing the open-source Model Context Protocol to give AI Agents standardized, safe read/write access to external systems.

  • Infrastructure Abstraction: MCP Servers act as isolated sandboxes. The "Kubernetes MCP" exposes specific SDK wrappers allowing the Fixer Agent to restart failed pods or scale deployments dynamically.
  • Human-in-the-loop (HITL): Hardcoded execution halts. If the Correlator Agent detects a SEV-1 incident, the workflow pauses and invokes a Slack Webhook. The pipeline is suspended utilizing asyncio.Event until human authorization via Slack API triggers the pipeline to resume.
5. Infrastructure & DevOps

Containerization & Resilience

Docker Docker Compose Zookeeper (Kraft) Chaos Engineering

Microservice Orchestration

The entire platform is defined as Infrastructure as Code (IaC) via a master docker-compose.yml.

  • Network Topology: A unified Docker bridge network allowing high-speed internal DNS resolution between the Node.js Producer, Kafka Broker, Python Consumer, PostgreSQL Database, and Qdrant Vector DB.
  • Persistent Volumes: Docker named volumes (postgres_data, qdrant_data) to ensure data permanence across container destruction and recreation lifecycles.

Testing & Chaos Engineering

The system is designed to handle failure, making it highly testable and robust for production pipelines.

  • CI/CD Integration: Github Actions pipeline executing standard Python Flake8 linting, Node.js ESLint, and unit tests upon Pull Request generation.
  • Simulated Disaster Recovery: Utilizing Chaos Engineering principles to intentionally destroy internal Docker containers during runtime to test the NightOwl agent's ability to ingest the resulting Grafana alert, diagnose the failure, and autonomously execute a reboot script via MCP tooling entirely unassisted.
